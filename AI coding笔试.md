例题1：

```python
# 推理网关

## 题目目标
实现一个面向大模型推理场景的 HTTP 网关，支持：
- 智能路由
- 负载均衡
- KV Cache 感知调度

关键挑战：在“提升缓存命中率”和“防止单机过载”之间做平衡。

## 场景说明
相同前缀请求若路由到同一后端，可复用 KV Cache 并降低延迟；但过度聚合会导致热点与排队。
你需要实现一个带策略权衡的调度器，而不是纯轮询。

## 简化假设
- 后端模型服务可使用 Mock
- Prefix 匹配可用字符串前缀
- 负载可用并发请求数近似

## 功能要求（必做）
1.  **网关基础能力**
    - 实现 HTTP 服务器，可接收 OpenAI 风格的 `/v1/chat/completions` 请求
    - 支持配置多后端实例（每个后端有地址、当前并发数、模型信息）
    - 能将请求转发到选中的后端，并返回响应给客户端

2.  **路由策略**
    - 基于请求 Prompt 的前缀，优先路由到已有相同/相似前缀请求的后端（提升 KV Cache 复用率）
    - 当多个后端都有匹配前缀时，选择其中负载最低的
    - 当无匹配前缀时，按负载均衡策略选择后端（如轮询/最小连接数）

3.  **负载与健康管理**
    - 记录每个后端的当前并发请求数，拒绝超过阈值的请求
    - 支持后端健康检查（可简化为简单的 Ping 接口）
    - 当后端负载过高时，自动降低该后端的路由权重

4.  **可观测性（加分项）**
    - 记录每个请求的路由决策日志（选择的后端、原因、耗时）
    - 提供简单的 Prometheus 指标（如请求总数、缓存命中率、各后端负载）
```
