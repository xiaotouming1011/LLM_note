```markdown
访问网站完整全流程（超通俗版，一步不落）
1. 输入域名 → DNS 域名解析
你在浏览器输入网址（如www.baidu.com），域名只是别名，电脑只认 IP。
浏览器先查本地缓存、系统 hosts 文件
没有就向DNS 服务器发请求
DNS 把域名翻译成服务器公网 IP 地址
2. 建立 TCP 三次握手
浏览器拿到 IP 后，和网站服务器建立TCP 连接（可靠传输）：
你发：我要连你（SYN）
服务器回：收到，同意连（SYN+ACK）
你再回：我收到你的同意（ACK）
👉 连接建立成功
3. HTTPS TLS 握手（加密通道）
现在大多是 HTTPS，在 TCP 基础上做加密：
服务器下发SSL/TLS 证书
浏览器验证证书是否合法、是否过期
双方协商加密算法、生成会话密钥
👉 建立加密安全通道，防止被窃听、篡改
4. 发送 HTTP/HTTPS 请求
浏览器组装HTTP 请求报文：
请求行（方法 GET/POST + 路径 + 协议版本）
请求头（浏览器标识、Cookie、缓存策略、语言等）
请求体（POST 才会有，传表单 / JSON 数据）
通过建好的加密通道发给服务器
5. 服务器处理请求
服务器端流程：
防火墙 / 负载均衡接收请求
转发到后端 Web 服务（Nginx/Apache）
交给业务程序（Java/PHP/Python/Node）
查数据库、调接口、处理业务逻辑
组装要返回的页面 / 数据
6. 服务器返回 HTTP 响应
服务器返回响应报文：
状态码（200 成功、404 找不到、500 服务器错）
响应头（缓存规则、内容类型、Cookie 等）
响应体（HTML、CSS、JS、图片、接口 JSON等资源）
7. 浏览器断开 TCP 四次挥手
数据传输完，双方礼貌断开连接：
四次挥手互相确认关闭，释放端口资源
8. 浏览器渲染页面
浏览器拿到 HTML 后：
解析 HTML 生成DOM 树
解析 CSS 生成CSSOM 树
合成渲染树
布局→绘制→回流重绘
加载并执行 JS、渲染图片视频
最终展示出网页
一句话极简串：
域名 DNS 解析→TCP 三次握手→TLS 加密握手→发 HTTP 请求→服务器业务处理→返回响应→四次挥手断连→浏览器渲染页面


```
