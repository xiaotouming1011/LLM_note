# Linux 问题排查「缺乏体系」到底啥意思+怎么练成体系化
## 一、别人说你缺体系，具体指哪些表现
1. **东一榔头西一棒子**：乱敲命令，想到啥查啥，没有固定流程，全靠蒙。
2. **只看表象不溯源**：报错就搜报错，不看日志、不看资源、不看链路，找不到根因。
3. **不会分层排查**：分不清是**硬件/系统内核/网络/进程/服务/配置/应用**哪一层的问题。
4. **只会用现成命令，不懂原理**：知道 `top、df、ping`，但不会解读指标、不会联动分析。
5. **排查完没复盘**：解决了就完事，不总结套路，下次遇到同类问题还是从头瞎查。

---

## 二、标准体系化排查总思路（记住这个万能框架）
**从「先看整体 → 再分层定位 → 最后精准抓根因」**
固定顺序：
1. **先看全局状态**（整机负载、资源瓶颈）
2. **再分三大维度**：CPU / 内存 / 磁盘IO / 网络
3. **然后查进程&服务**（谁占资源、谁挂了、谁僵尸）
4. **查日志**（系统日志、应用日志、中间件日志）
5. **查配置&权限&端口**
6. **最后链路验证**（本地→网络→依赖→应用）

---

## 三、给你一套可直接背的 Linux 排查标准流程（体系模板）
### 1. 系统整体概览（先定有没有瓶颈）
```bash
uptime        # 负载、开机时间
top/htop      # 全局进程资源
free -h       # 内存
df -h / du -sh# 磁盘空间
```
**先判断**：是CPU爆了、内存溢出、磁盘满了、还是IO卡死。

### 2. 分层定位四大资源
- **CPU**：top 看 %CPU、看哪个进程占用高，再看是不是死循环、频繁GC、上下文切换
- **内存**：free、swap 是否占用过高，OOM 有没有杀进程
- **磁盘**：inode满？磁盘使用率100%？iowait 高不高
- **网络**：ping、telnet、ss -lntp、防火墙、端口监听、丢包、延迟

### 3. 进程与异常排查
- 僵尸进程、僵死、崩溃重启
- 后台进程无故退出、端口被占用
```bash
ps -ef | grep xxx
ss -lntp
kill -9 谨慎用
```

### 4. 日志是根因核心
系统日志：`/var/log/messages`、`dmesg`
应用日志：项目日志、nginx、redis、mysql 日志
**体系化关键：先看日志报错，再反向定位资源和配置**

### 5. 配置 & 权限 & 环境
权限不足、SELinux、防火墙、环境变量、配置文件写错、版本兼容问题

### 6. 链路闭环
本机没问题 → 查内网 → 查端口 → 查依赖服务 → 查第三方接口

---

## 四、怎么快速改掉「没体系」，练成专业排查
1. **固定排查顺序**：以后不管什么问题，强制按上面流程走，不许乱敲命令。
2. **每个命令不只敲，要会看指标**：比如 load average、iowait、swap、cached 含义。
3. **归类问题模板**：
   - 机器卡慢模板
   - 端口连不上模板
   - 服务启动失败模板
   - 内存飙升OOM模板
4. **每次排完复盘**：记一句：根因是哪一层、用了哪几个命令、下次怎么快速定位。
